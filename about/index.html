About me

Hello there! My name is Giacomo Furlan!

Who I am

I’m a senior software engineer focused on web technologies, but not bound to them at the same time. I try to always learn something, move on, evolve myself from a personal and a professional point of view.

I’ve always been fascinated more than what is happening “under the hood”, rather than using tools, playing video games etc. For this reason I unwittingly started my professional carreer writing code for Neverwinter Nights, which I truly believe it’s one of the best videogames ever made! I soon wanted to share my works and built my own very first (and ugly :D) website, written in HTML and CSS: from then on, I’ve trained myself in a series of web technologies that took me here, now.

My skills

This is an uncomplete list of skills I master

Programming / scripting languages

  • PHP: first and foremost used scripting language in my skills list, always updating my knowledge following the releases (thus including PHP 7.1 and keeping updated on release candidates).
  • JavaScript: following PHP, JS is one my favourite languages, including ES6, experimental ES 2016/2017 standards (more to follow).
  • HTML/CSS stack: proficiency use of these two languages, following the latest standards.
  • Bash: to create from the simpliest to the most complex script helpers.
  • Go: new entry in this list, I’ve recently started working with this language, and I’m really enjoying returning back to compiled code.
  • MySQL, PostgreSQL, SQLite… I work with all of them.
  • MongoDB: mostly working with it via PHP (Doctrine ODM).

Languages I know, but I don’t develop with on a daily basis:

  • C and C++: university projects and a few ones in my professional carreer.
  • C#: roughly 6 months on a .net MVC project professionally, on personal projects before.
  • Java (learnt mostly writing Android projects).

Details

  • PHP: I usually develop projects written on Symfony (last version used at the time of writing: 3.4), bundled with a series a plugins like Doctrine 2 (ORM/ODM), interfacing with external services (Amazon SQS, RabbitMQ, MySQL, etc), writing backend algorithms to analyse data and writing as close as possible to PSR standards.
  • JavaScript: writing different kind of scripts for node applications, transpiled web interface logics or backward compatibility code for smaller files. Working with a series of tools, like yarn / npm, Webpack and Babel.js, using the newest standards and proposals (EcmaScript 6/2015, up to 2017 proposals), using linters (eslint) to maintain a code style, transpilers and frameworks (one over the others: React from facebook sometimes together with Redux).
  • Bash: in my professional carreer, I had the opportunity to write a series of bash scripts, from the simpliest to the most complex ones, for example to orchestrate the docker development environment, docker deploys, calling the test suites etc.
  • Go: mostly used for personal projects (see my GitHub page!), I’ve first learnt it for a tool that I had to write in order to send the (filtered and formatted) system logs to a Graylog instance.

Technologies

  • git: we all know what git is! Branching, rebasing, pull-requesting, cherry-picking and so on so forth.
  • Docker CE: I use it to create development environments as well as production ones, deploying docker images on clusters.
  • Graylog: to monitor the access log as well as tracking JS, PHP and server errors on the fly (downside: very big ElasticSearch database!).
  • Linux / Unix system tools (like systemd, logrotate, htop, dstat, fdisk…): I have no problems navigating in a unix system at all!
  • Amazon EC2: virtual servers, load balancers, auto-scaling groups etc.
  • RabbitMQ: AMQ server.

Editor / IDE of choice

  • Atom (editor): a powerful editor with a great community behind deploying new plugins everyday. My first choice to develop JS and go.
  • PHPStorm (IDE): from IntelliJ, the most complete IDE for PHP I’ve ever used. May be slow from time to time, so I keep using the lightweight Atom whenever possible.

Spoken and written languages
